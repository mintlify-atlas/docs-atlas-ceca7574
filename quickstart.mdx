---
title: "Quickstart"
description: "Get E-Dokan Store running locally in under 5 minutes"
---

<Note>
  E-Dokan Store is a frontend-only storefront. It requires a running backend API at a reachable URL before it will display any products, categories, or billboard content. Make sure your API is up and its base URL is ready before you begin.
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/hempun10/edokan-store
    cd edokan-store
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the project root and set the URL of your backend API:

    ```bash
    NEXT_PUBLIC_API_URL=https://your-api-url.com
    ```

    `NEXT_PUBLIC_API_URL` is the only required environment variable. Every data-fetching helper in `actions/` prepends this value to its request path.

    <Tip>
      The `NEXT_PUBLIC_` prefix makes the variable available in the browser as well as in server components, which is necessary for any client-side requests (e.g. the checkout flow).
    </Tip>
  </Step>

  <Step title="Start the development server">
    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
  </Step>

  <Step title="Open the storefront">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser.

    You should see the home page with the hero billboard and the **Featured Product** section populated with data from your API.
  </Step>
</Steps>

## Available scripts

All scripts are defined in `package.json` and invoked via your package manager of choice.

| Script | Command | Description |
|---|---|---|
| Development server | `npm run dev` | Starts Next.js in development mode with hot reload |
| Production build | `npm run build` | Compiles and bundles the app for production |
| Production server | `npm run start` | Serves the production build on port 3000 |
| Lint | `npm run lint` | Runs ESLint using `eslint-config-next` |

<Warning>
  Run `npm run build` before `npm run start`. Starting the production server without a build will fail.
</Warning>
